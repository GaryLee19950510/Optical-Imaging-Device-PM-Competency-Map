Section 01

SoC 全局架构 — 影像产品的"大脑"

SoC 不是一颗"芯片"，而是一个"平台"——选 SoC 本质是选平台生态，不是选参数最高的硅片

🧠 SoC = 整合平台

SoC（System on Chip，片上系统）将 CPU、ISP、VPU、NPU、GPU、DSP、内存控制器、各种接口和安全引擎等所有核心功能集成在单一芯片内。影像产品的 SoC 不仅仅是"跑系统"的应用处理器——它是影像链路的心脏，决定了产品能做到什么、做不到什么。

🎯
PM视角选 SoC = 选"开发平台 + 生态 + 供应链"三合一。芯片参数只是表面，SDK 成熟度、IQ 工具链、FAE 响应速度才是隐性成本大头。
🔄 SoC 集成 vs 分立方案

集成 SoC 方案（如 Ambarella CV2）：ISP/VPU/NPU 全在一颗芯片内，走线短、功耗低、开发快，但灵活性受限于芯片厂商的 ISP 能力和 AI 模型支持。

分立方案（如 FPGA + 独立 ISP + 独立 NPU）：灵活度极高，可定制每一环，但 BOM 成本高、功耗大、开发周期长，适合工业/军用。

消费级影像产品几乎全部选择集成 SoC，因为成本和开发周期的压力远大于定制需求。

Section 02

ISP 流水线详解 — 从 RAW 到美景的 15 级旅程

ISP 的工作就是把 Sensor 输出的"生肉"RAW 数据处理成用户可看的"佳肴"YUV 帧

📷 Sensor = 摄影师

Sensor 负责采集光信号，输出 Bayer RAW 数据——每个像素只有一种颜色（R/G/B），图像是"马赛克"状态，看起来像没有上色的线稿。RAW 数据暗、噪、偏色、暗角，完全不适合直接观看。

🎨 ISP = 后期调色师

ISP 接手 RAW 数据后，经过 15 级流水线处理：补坏点、去暗角、还原色彩、降噪、HDR 合成、锐化……最终输出干净、色彩准确、人眼舒适的 YUV 图像。同一颗 Sensor，不同 ISP 调出来的画质可以天差地别。

01 黑电平校正
Black Level Correction (BLC)
Sensor 在完全黑暗时输出的信号并不是 0，而是一个非零的偏置电平（黑电平），由暗电流和电路偏移引起。BLC 的作用就是减去这个基底，让黑色区域真正归零。
BLC 校准不准会导致画面整体偏亮或偏暗。选型评审时关注 SoC 是否支持 BLC 自动校准，以及校准精度。
🔴
02 坏点校正
Defect Pixel Correction (DPC)
Sensor 在制造过程中会产生少量坏像素（始终输出最大值的"亮点"或始终输出 0 的"暗点"）。DPC 通过检测和插值替换这些坏点，避免画面出现刺眼的亮点或黑点。
坏点数量随 Sensor 老化会增加。量产 DVT 阶段要做坏点统计，超过规格的 Sensor 需退货。
🌘
03 镜头阴影校正
Lens Shading Correction (LSC)
镜头光学特性导致画面边缘进光量少于中心（暗角），且边缘可能出现色偏。LSC 通过二维增益表对不同位置像素施加不同补偿，让整帧亮度和色彩均匀。
LSC 校准需要在 IQ Tuning 阶段用均匀光源拍摄标定。换镜头必须重新标定 LSC，否则暗角明显。
🧩
04 去马赛克
Demosaic / Bayer Interpolation
Bayer RAW 中每个像素只有 R/G/B 中的一个通道。Demosaic 通过插值算法，利用相邻像素的颜色信息，为每个像素补齐另外两个通道，还原完整的 RGB 图像。插值算法质量直接影响色彩伪影和摩尔纹。
好的 Demosaic 算法能减少摩尔纹和色彩伪影，这是 ISP 差异化的关键能力之一。低价 SoC 的 Demosaic 算法通常较简单，在高频纹理（如栅栏、布料）场景容易出现伪影。
⚖️
05 自动白平衡
Auto White Balance (AWB)
不同光源（日光 6500K、白炽灯 2800K、荧光灯 4000K）的色温不同，同一物体在不同光源下呈现不同颜色。AWB 自动识别场景色温，动态调整 RGB 增益比，使白色物体在任何光源下都保持白色。
AWB 算法质量直接决定"偏色"投诉率。常见的 AWB 问题：室内偏黄/偏绿、混合光源失灵。选型时要求供应商提供复杂光源（如日光+白炽灯混合）的 AWB 实测效果。
☀️
06 自动曝光
Auto Exposure (AE)
AE 通过分析画面亮度直方图，自动调整曝光时间（快门速度）和模拟增益（ISO），维持画面亮度在目标水平。AE 收敛速度和稳定性是核心指标——过慢导致切换场景时画面"一暗一亮"，过快则产生"呼吸效应"。
AE 收敛时间 ≤ 0.5s 是消费级基线。运动相机需要更快收敛（≤0.3s），监控相机可以稍慢但更稳定。DVT 阶段必须测试快速切换场景（如从室内走到室外）的 AE 表现。
🔍
07 自动对焦
Auto Focus (AF)
AF 通过评价画面清晰度（对比度法 / 反差对焦，或 PDAF 相位检测对焦），驱动镜头的音圈电机（VCM）移动到最清晰位置。PDAF 对焦速度更快（1帧内判断方向），但需要 Sensor 支持相位检测像素。
固定焦距的监控/门铃不需要 AF，但 Vlog 相机、云台相机必须有 AF。选型时确认 SoC AF 算法支持 PDAF + 反差对焦混合方案，以及 AF 区域模式和速度档位。
🎨
08 色彩校正矩阵
Color Correction Matrix (CCM)
每个 Sensor 的光谱响应曲线都不同，导致同样场景下颜色偏移。CCM 用 3×3 矩阵将 Sensor 的色彩空间映射到标准 sRGB 空间，修正固有色偏。CCM 需要在 IQ Tuning 阶段用标准色卡（如 X-Rite ColorChecker）标定。
CCM 标定不准导致肤色偏绿/偏红，是用户投诉的高频问题。IQ 调优时要求重点验证肤色还原。
📈
09 Gamma 校正
Gamma Correction
Sensor 输出的光信号是线性的（光强×2 → 信号×2），但人眼对亮度的感知是非线性的（对数响应）。Gamma 校正将线性信号映射为非线性编码（γ≈2.2 或 sRGB 曲线），使暗部细节更丰富、亮部不过曝，符合人眼的感知习惯。
Gamma 曲线是可以调整的——这就是为什么不同手机拍同一场景"风格"不同。有些厂商会定制 Gamma 曲线来获得特定的"电影感"或"真实感"。
🔇
10 降噪 (2D/3D NR)
Noise Reduction (2DNR / 3DNR)
2DNR（空间降噪）：在单帧图像内，利用像素邻域的相似性消除随机噪点，但会损失细节。3DNR（时域降噪）：跨帧比对，将帧间变化小的区域判定为噪声并消除，低照度效果显著优于 2DNR，但运动物体可能出现拖影（Ghost）。
3DNR 是夜景画质的分水岭——好的 3DNR 既降噪又保细节、运动拖影极少。这是各 SoC 厂商的"看家本领"，选型时必须在低照+运动场景实测对比。3DNR 强度可调，量产时需根据场景平衡"干净度"和"拖影感"。
🌗
11 HDR 合成
High Dynamic Range (HDR)
单帧曝光无法同时覆盖极亮和极暗区域。HDR 方案是拍摄短/中/长多帧不同曝光时间的图像，然后对齐融合，在一帧中同时保留亮部细节和暗部细节。关键技术挑战是运动对齐（Motion Compensation），否则运动物体会产生鬼影。
运动相机的 HDR 质量直接决定逆光场景可用性。选型时必测"强逆光+运动物体"场景。HDR 模式通常会降低帧率（需多帧合一），需在产品规格中明确 HDR 开启后的帧率。
🌐
12 宽动态
Wide Dynamic Range (WDR)
WDR 是安防监控领域的惯用术语，本质也是扩展动态范围，但通常指单帧内的双斜率/多斜率曝光方案（一行像素在同一帧内经历两种不同曝光时间），而非多帧融合。WDR 不依赖多帧对齐，因此不会产生运动鬼影，但动态范围扩展能力弱于多帧 HDR。
监控/门铃类产品优先选 WDR（无鬼影），运动/Vlog 相机优先选多帧 HDR（动态范围更大）。选型时确认 SoC 支持哪种方案。
🔪
13 边缘增强/锐化
Sharpening / Edge Enhancement
Demosaic 和降噪过程会损失一部分高频细节，使图像看起来"软"。锐化通过增强边缘对比度来提升主观清晰感。但过度锐化会产生振铃（Halo）和噪声放大，需要平衡。
锐化是双刃剑——锐化不够画面"糊"，锐化过度边缘出现白边/振铃。这个参数在 IQ Tuning 中需要反复调试，且不同场景（白天 vs 夜景）需要不同锐化强度。
🌈
14 色彩增强/饱和度
Color Enhancement / Saturation
在 CCM 和 Gamma 之后，对色彩的饱和度和色相进行主观调优，使天空更蓝、草地更绿、肤色更自然。这是"风格化"处理，各品牌的产品"色彩风格"差异主要来自这一步。
色彩增强是品牌差异化的手段。需要与产品经理（产品定义方）确认色彩风格偏好——"真实还原"还是"讨喜增强"？这个决策影响 IQ 调优方向。
✂️
15 缩放/裁切
Crop & Scale / Digital Zoom
将 ISP 输出的图像裁切（Crop）或缩放（Scale）到目标分辨率。EIS 电子防抖需要裁切画面（留出稳定余量），数字变焦也是通过裁切+放大实现。裁切后有效像素减少，画质下降。
EIS 裁切通常占画面 10~15%，意味着 Sensor 需要额外的分辨率余量。4K + EIS 的产品，Sensor 实际需要超采样（如 4K+ 的 Sensor 裁切后输出 4K）。选型时必须把 EIS 裁切计算在内。

3A 联动 — AE / AWB / AF 如何协同

3A 联动时序关系
🎯
3A 联动的 PM 排查方向
① 画面一暗一亮→ AE 收敛慢或增益跳变，调 AE 收敛速度和步长
② 偏色随场景切换→ AWB 跟随滞后，需优化 AWB 算法响应速度
③ 对焦拉风箱→ AF 收敛策略不稳定（过冲+回拉），调 AF 评估函数和搜索策略
④ 夜景对焦失败→ 低照度对比度太低，AF 无法判断清晰度方向，需要辅助对焦灯

ISP 的硬件实现方式

🔧 硬件流水线（固定功能）

每级 ISP 处理由专用硬件电路完成，延迟极低、功耗极低，但功能固定不可编程。大部分消费级 SoC 的 ISP 采用此方式。优点是稳定高效，缺点是换 Sensor 后调优空间受限于硬件能力。

💻 可编程 DSP/FPGA

ISP 算法运行在可编程处理器上，灵活度极高，可以随时更新算法，但功耗高、延迟大。高端专业设备（如电影摄影机）可能采用此方案。消费级极少使用。

🔀 混合架构（主流）

大部分硬件流水线 + 关键级可编程（如 3A 算法、降噪参数可配置），兼顾效率和灵活性。这是目前消费级影像 SoC 的主流方案。PM 应关注：哪些级可编程、可编程范围有多大。

IQ 调优（Tuning）— 从能看变好看

🎯 什么是 IQ Tuning

为特定的 Sensor + 镜头组合，定制 ISP 的所有参数（3A、降噪、CCM、Gamma、锐化等），使画质在该硬件组合下达到最优。换一颗 Sensor 或换一个镜头，原则上都需要重新 Tuning。

⚠️
Tuning 周期通常需要 2~4 个月，涉及大量场景拍摄和参数迭代。这是项目进度中的关键路径，PM 必须提前规划。
📋 Tuning 流程

① 场景定义：确定需要覆盖的关键场景（室内/室外/夜景/逆光/人像/运动）
② 拍摄标定：在标准光源箱中拍摄色卡、灰卡，标定 LSC/CCM/BLC
③ 3A 调优：调试 AE/AWB/AF 参数，确保收敛速度和稳定性
④ 降噪/锐化调优：在各类场景中平衡干净度和细节
⑤ 主观评审：团队主观评价，迭代调参
⑥ 量产固化：参数写入固件，锁定版本

Section 03

数据流与互连 — SoC 内部如何搬运数据

影像系统最大的隐形瓶颈不是算力，而是数据搬运的带宽

MIPI-CSI：Sensor 与 SoC 之间的"高速公路"

🔌 MIPI CSI-2 协议基础

MIPI CSI-2（Camera Serial Interface 2）是 Sensor 与 SoC 之间的高速串行差分接口标准。

核心参数：
• Lane 数：1/2/4 Lane，Lane 越多带宽越大
• 单 Lane 速率：最高 2.5 Gbps（CSI-2 v3.0 可达 4.5 Gbps）
• 虚拟通道：同一物理 Lane 可复用传输多路数据

带宽计算：
4K30 RAW10 = 3840×2160 × 30 × 10bit ≈ 2.5 Gbps
需要 2-Lane × 1.5Gbps 或 4-Lane × 800Mbps

📷 双摄/三摄 MIPI 复用

当产品需要多个 Sensor（如双目追踪、鱼眼+长焦），SoC 的 MIPI-CSI 通道数成为硬约束。

复用策略：
• 分时复用：多个 Sensor 轮流使用同一组 Lane，切换有延迟
• 独立通道：SoC 提供 2~3 组独立 CSI 接收器，无干扰但成本高
• CSI 聚合器：外部芯片将多路 Sensor 汇聚到一路 CSI

PM 注意：双摄产品在选型时必须确认 SoC 有≥2 组独立 CSI 接收器，否则只能分时复用，双路同时预览会丢帧。

⚠️
接口 Mismatch = 点不亮选型时 Sensor MIPI Lane 数和速率必须与 SoC CSI 接收能力匹配，否则无法点亮或帧率不达标。这是 HW/SW 协同评审必查项。

内存带宽 — 影像系统的隐形瓶颈

ISP 处理每一帧图像时，RAW 输入 → DDR → ISP 各级处理 → DDR → VPU 编码 → DDR → 码流输出，一帧图像在 DDR 中至少被读写 3~5 次。当 ISP + VPU + NPU 同时运行，DDR 带宽会被快速吃满。

DDR 带宽不足的典型表现
① 丢帧：ISP 输出帧率不稳定，偶发掉帧
② 编码码率波动：VPU 拿不到稳定帧源，输出码率忽高忽低
③ AI 推理延迟：NPU 等待 DDR 数据，推理时间从 30ms 暴涨到 100ms+
选型时预留 30% DDR 带宽余量，是行业经验的安全线。

VPU 编码通路

🎬 硬件编码 vs 软件编码

硬件编码（VPU）：专用电路完成 H.264/H.265 编码，CPU 占用接近 0%，延迟约 1~2 帧（30~60ms），功耗极低。消费级影像产品必须使用硬件编码。

软件编码（CPU）：用 CPU 跑编码算法，灵活但占用大量 CPU 和功耗，延迟高，仅用于调试或特殊格式。

📊 码率控制策略

CBR（固定码率）：码率恒定，适合带宽受限（如云端存储），但复杂场景画质下降。

VBR（可变码率）：按内容复杂度动态调整码率，静止场景省码率，运动场景保画质，但峰值码率可能超出预算。

AVBR（自适应可变码率）：在 VBR 基础上增加上限约束，兼顾质量和带宽预算。

PM 选择：云端存储选 CBR/AVBR，本地存储选 VBR。

供应商选择的三维权衡
① 技术匹配度：ISP 能力 + NPU 算力 + 接口组合是否满足产品需求
② 生态匹配度：SDK 成熟度 + IQ 工具链 + 社区资源 = 开发效率

经验法则：生态匹配度往往是最终决策的"胜负手"——芯片参数再好，SDK 烂、FAE 慢、Tuning 工具难用，项目一样延期。
⚠️
FAE 支持力度的隐性成本
FAE（Field Application Engineer）是原厂派驻的技术支持。FAE 响应速度直接决定 IQ Tuning 周期和 Bug 修复速度。

谈判要点：
• EVT 阶段是否派 FAE 驻场？驻场多久？
• IQ Tuning 是否包含在 SoC 采购合同中？
• SDK Bug 修复的 SLA（响应时间）是多少？
• 是否有参考设计（Reference Design）可直接用？
ℹ️
二供策略：对于关键产品，应评估是否需要 Pin-to-Pin 或软件兼容的替代 SoC 方案。影像 SoC 通常没有真正的 Pin-to-Pin 替代（不像 Sensor 有 SmartSens 替代 Sony），但可以准备软件架构层抽象——通过 HAL（硬件抽象层）隔离 SoC 差异，降低切换成本。

PM 排查思路三步法

① 定位环节

Sensor 问题 vs ISP 问题 vs 镜头问题

• 换 Sensor 同款不同批 → 问题消失 = Sensor 问题
• 调 ISP 参数后问题改善 = ISP 问题
• 换镜头后问题改善 = 镜头问题

不要上来就改 ISP 参数——先确定问题归属。

② 缩小范围

做 A/B 对照实验

• 换 Sensor → 对比画质
• 换镜头 → 对比画质
• 调参数（一次只改一个） → 对比画质
• 不同光照条件 → 对比画质

每次只改变一个变量，否则无法定位根因。

③ 推动闭环

把 IQ 问题翻译成供应商可执行的调优任务

✗ "画质不好"（供应商无法执行）
✓ "室内 300Lux 混合光源下，AWB 偏绿 ΔE=8.5，目标 ΔE<5"（可执行）

提供标准化的复现条件、量化指标和目标值，才能推动供应商高效闭环。

Section 08

行话速查表 — SoC/ISP 相关术语

开会时你会遇到的所有缩写，按类别整理，拒绝被术语"降维打击"

🧠 SoC & 系统类

片上系统 SoC
System on Chip。把 CPU/ISP/NPU/VPU 等都塞进一颗芯片，是影像产品的"大脑平台"。
图像信号处理器 ISP
Image Signal Processor。把 Sensor 的"生肉"RAW 处理成"佳肴"YUV 的专用硬件模块。
视频处理单元 VPU
Video Processing Unit。专用视频编解码硬件，支持 H.264/H.265/MJPEG。
神经网络处理单元 NPU
Neural Processing Unit。专门跑 AI 推理的硬件加速器，算力单位 TOPS。
万亿次运算/秒 TOPS
Tera Operations Per Second。NPU 算力单位，1 TOPS = 每秒 1 万亿次运算。注意实测值通常低于标称值。
双倍速率内存 DDR
Double Data Rate SDRAM。SoC 外挂的运行内存。DDR 带宽是影像系统的隐形瓶颈。
可信执行环境 TEE
Trusted Execution Environment。SoC 内的安全区域，用于运行安全敏感代码（如密钥存储、DRM）。
硬件抽象层 HAL
Hardware Abstraction Layer。隔离 SoC 差异的软件中间层，方便切换 SoC 平台。

🎨 ISP & 图像处理类

3A 自动控制 AE+AWB+AF
自动曝光 + 自动白平衡 + 自动对焦。ISP 中最重要的三个自动控制系统，联动工作。
图像质量 IQ
Image Quality。综合评价画质好坏的指标体系，涵盖清晰度、色彩、噪声、动态范围等。
ISP 调优 Tuning
为特定 Sensor+镜头组合定制 ISP 参数的工程工作。每换一颗 Sensor 或镜头都需重新 Tuning。
去马赛克 Demosaic
把 Bayer 阵列的稀疏色彩数据插值成每像素 RGB 三通道的完整彩色图像。
宽/高动态范围 WDR/HDR
扩展动态范围，同时保留亮部和暗部细节。WDR 通常指单帧方案，HDR 指多帧融合。
黑电平校正 BLC
Black Level Correction。补偿传感器暗场偏移，让黑色归零。
镜头阴影校正 LSC
Lens Shading Correction。补偿画面四角亮度下降和色偏。
色彩校正矩阵 CCM
Color Correction Matrix。3×3 矩阵将 Sensor 色彩映射到标准 sRGB。
降噪 NR
Noise Reduction。2DNR = 空间域单帧降噪；3DNR = 时域跨帧降噪，低照效果更好但可能拖影。
电子防抖 EIS
Electronic Image Stabilization。通过裁切画面+陀螺仪数据实现视频稳定，不需要物理云台。

📡 接口 & 编码类

相机串行接口 MIPI-CSI
Camera Serial Interface。Sensor 和 SoC 间的高速串行接口，支持 1~4 Lane 差分传输。
原始图像数据 RAW
Sensor 未经 ISP 处理的原始 Bayer 输出，每像素只有一个颜色通道。
亮色分离格式 YUV / YCbCr
亮度和色度分离的图像格式，ISP 最终输出格式。Y=亮度，U/V=色度。
现场应用工程师 FAE
Field Application Engineer。原厂派驻客户现场的技术支持，ISP Tuning 阶段的关键资源。

💰 供应链 & 项目类

软件开发套件 SDK
Software Development Kit。SoC 厂商提供的驱动、库、工具和文档集合，是开发的基础。
参考设计 Reference Design
SoC 厂商提供的完整硬件设计（原理图+PCB），可直接参考或修改，大幅降低开发难度。
长期协议 LTA
Long Term Agreement。与供应商签订的长期供货协议，锁定产能和价格，降低断供风险。
第二供应商 Second Source
除主供应商外的备份供应源，降低单一来源风险。影像 SoC 通常无 Pin-to-Pin 替代。
